# 覆盖审计(coverage audit)

把"知识地图有没有重大遗漏"从 agent 自述,变成**可量化、可复现的客观校验**。本文是 audit 行为的**单一事实源**;可执行实现见 `templates/audit.py.tmpl`。

## 1. 定位:audit 是什么、不是什么

- **是后置校验工序,不是 adapter。** adapter(`adapters.md` 的 `probe()`+`extract()`)从代码提取图数据;audit 的**输入是 AI 已产出的 `graph.json` + 文档**,校验它们有没有结构性遗漏。audit 不 probe、不 extract、不理解代码。
- **零外部依赖,永远可跑。** 只用 `os`/`subprocess` 调 `os.listdir`/`find`/`wc`/`grep` 等标准工具。任何环境(内网/离线/私有仓)都能跑——这是它当客观标尺的前提。
- **执行位置:** SKILL.md 执行流程的 Step 5(自检)。并发模式下由**整合 agent 串行**跑(归并 graph 之后、产 report.md 之前),**绝不并发**(见 `parallel-extraction.md`)。

## 2. 核心信念:职责分离 + 可靠性分层(诚实边界)

**职责分离**——AI 和命令各做各擅长的,互相校验:

| 工序 | 谁做 | 为什么 |
|---|---|---|
| 理解代码(聚类成子系统、画业务流程、判定显著) | AI(生成阶段) | 需要语义,命令做不到 |
| 穷举校验(AI 的理解有无结构遗漏、cite 真不真) | 命令(校验阶段) | 需要穷举,AI 会漏 |

> **不是让命令去理解代码(它做不到),也不是让 AI 去穷举(它会漏)。** AI 会因注意力漏掉整个子系统(Hermes 实测漏了 datagen),命令穷举不会。

**可靠性分层(必须诚实标注,不假装全知):**

| 层 | 标尺 | 能否保证零遗漏 |
|---|---|---|
| **结构层**(覆盖/cite/规模) | 文件系统客观测量 | ✅ **能**(闭集,穷举) |
| **语义层**(职责/流程/隐式机制/业务正确性) | 无客观标尺 | ❌ **不能**,只能多策略 grep 逼近 + 透明留白 |

**audit 只承诺结构层。** 语义层(某业务流程画得对不对、某隐式机制讲清没)没有客观标尺,任何工具(含 Greptile/Serena/纯 AI)都保证不了零遗漏——这是代码知识地图的天花板,不是本 skill 的缺陷。audit 的本分是把结构层锁死,把语义层的未知**写在 report 脸上**(列出"搜了哪些策略""哪些已知未深入")。

## 3. A 关 — 覆盖完备(核心关,无孤儿 = 无重大遗漏)

**标尺原则(本关灵魂):** A 关标尺**必须是文件系统客观测量**——`os.listdir(仓库根)` 取全量顶层实体(文件+目录),`find` 递归补充。**绝不能是 AI 产物**(graph.json、文档、README、AGENTS.md 都不能当标尺)。

> **铁证(为何标尺不能是 AI 产物):** Hermes 实测中 `run_agent.py` 实测 5679 行,但 graph.json 和 INDEX.md 都抄 `AGENTS.md` 写"~12k"——**两个 AI 产物对账一致,却都错了一倍**。拿 AI 产物当标尺,标尺本身被污染,校验毫无意义。只有不经过 AI 的文件系统能当绝对标尺。

**覆盖集抽取:** 从 graph.json 每个节点的 `files`(见 `graph-model.md`)+ `cite` + `path` 三处并集,收集所有被 claim 覆盖的顶层实体。

**判定规则(降到 cite 级,防"名义覆盖"):** 一个顶层实体算"已覆盖",**不是**"文件名出现在某节点 `files` 数组里"——那是弱判定(AI 可能塞进 `files` 充数却正文一字未提)。**真覆盖 = 有真 cite 指向它**(`file://路径#行号`,且 C 关验真通过)。只挂在 `files`/`path` 而无真 cite 的,标为**弱覆盖**(警告,不计入通过)。

**三态判定:** 文件系统每个顶层实体 →
- (a) 在覆盖集(有真 cite)= **已覆盖** ✅
- (b) 在 EXCLUDED(附理由)= **显式排除** ⚠️
- (c) 两者皆非 = **孤儿** 🔴(结构性遗漏,A 关失败)

**通过条件:** 孤儿数 = 0。

## 4. EXCLUDED 机制(显式排除,不能静默漏)

并非所有顶层实体都该进知识地图(测试套件、第三方依赖、i18n 资源、生成代码)。但**排除必须显式声明,不能静默漏掉**——静默漏掉与"遗漏"无法区分。

- **外置配置:** 目标仓库的 `.knowledge-map/audit-excluded.yaml`(**在目标仓库,不进 skill 仓库**)。格式极简(手写解析,免 PyYAML):
  ```yaml
  # 每行一个排除项,path 与 reason 以 ": " 分隔;reason 必填
  vendor/: 第三方依赖,非业务代码
  node_modules/: 前端依赖
  *_test.go: 测试套件,知识地图不纳入
  *.lock: 锁文件
  ```
- **reason 必填:** 无 reason 的排除项视为**无效,降级成孤儿**。强迫为每个排除给出理由,避免"图省事全排除"。
- **agent 维护,脚本只读:** audit 只读 excluded 配置,不写。生成知识地图时,主 agent 据该仓库实情产出 excluded 清单(可由模式候选 `README*`/`LICENSE`/`*.lock`/`*.toml`/`node_modules`/`vendor` 自动归类 + 人工确认)。

## 5. B 关 — 计数一致(辅助关,明确降级)

**比对三方:** `graph.json` 节点计数(按 kind) vs `INDEX.md`/`report.md` 文本里陈述的计数("N 个子系统""L3 N 篇") vs `wc`/`find` 实测的源文件数。

**降级声明(必须写进 report):** B 关**只抓"不一致",抓不到"共同遗漏"**。再引铁证:graph 和文档都抄同一错误数字,对账一致但都错——B 关对此**无能为力**。

- B 关**失败** = 计数对不上 → 抓的是真不一致(如 graph 有 10 节点、INDEX 写"9 子系统")。
- B 关**通过 ≠ 数字正确** —— 只是没有内部矛盾。

故 B 关定位为**辅助**:它抓"修了一半"(补了 graph 没补文档),抓不到"完全没发现"(那要靠 A 关)。**A 关是核心,B 关是辅助。**

## 6. C 关 — cite 验真

实读每个 cite 指向的文件,验:
- (a) 路径存在(`os.path.exists`);
- (b) 行号范围 `[L起, L止]` 落在文件实际行数内(`wc -l`);
- (c) 范围非空(`L起 ≤ L止`)。

纯结构校验,不验内容语义(不判断 cite 指向的代码"是不是真的相关"——那是语义层,无客观标尺)。

> 本关 = 复用并升级 `parallel-extraction.md` 既有的"cite 自检脚本化"(确定性脚本验证所有 `file://` 链接),把它统一进 audit。

## 7. 规模真实(子项,贯穿 A/B)

任何规模数字(行数、文件数、模块数)出现在 `graph.json`/`INDEX.md`/`report.md` 中,**必须由 `wc -l` / `find ... | wc -l` 实测得出**,禁抄 AI 文档(`AGENTS.md`、`README`、官网)。

audit 对所有规模断言重新实测,不符即报。这是从 12k 铁证直接长出来的规则:**抄来的数字不可信,只有文件系统实测可信。**

## 8. 退出码门禁(产出前置条件)

- **退出码 0** = A 关孤儿清零 **且** C 关 cite 全过(规模真实贯穿其中)→ 知识地图结构层可信,可交付。
- **退出码 1** = 任一核心关失败 → `report.md` 必须列出:孤儿清单 / 弱覆盖清单 / 失效 cite / 计数不一致 / 规模不符项。

SKILL.md §5 把"audit 退出码 0"作为**产出前置门禁**:非 0 不交付,回 `iterative-refinement.md` 收敛闭环(见 Phase 4)。

## 9. 参数化要求(衔接 `templates/audit.py.tmpl`)

audit 脚本**绝不硬编码仓库路径**——Hermes 版 audit.py 硬编码绝对路径 + Hermes 专属 EXCLUDED,换仓库即坏。必须参数化:

- **仓库根自动探测:** 从脚本位置或 `--repo` 向上找 `.git` 或 `.knowledge-map`。
- **命令行参数:** `--repo <path>` / `--km-dir <path>`(默认 `<repo>/.knowledge-map`) / `--excluded <yaml>`(默认 `<km-dir>/audit-excluded.yaml`) / `--graph <path>`(默认 `<km-dir>/zh/meta/graph.json`)。
- **EXCLUDED 外置:** 见 §4,不写死在脚本里。

这样换仓库时,skill 自动产出适配该仓库的 audit(读该仓库的 excluded、列该仓库的顶层实体),而非"脚本坏掉"。
