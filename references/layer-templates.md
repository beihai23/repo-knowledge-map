# 分层产出模板(L0–L3)

渲染每一层时照此结构产出。**业务流程图(时序/流程/状态)是代码知识库的核心价值,不是可选装饰**——新人最想看的是"这业务怎么跑起来的",而不是"模块怎么分的"。

## `<cite>` 与溯源格式

文档头部引用块:
```
<cite>
**本文档引用的源文件**
- [service/task/task_service.go](file://service/task/task_service.go)
</cite>
```

**每个图后**挂「图表来源」,**每个章节后**挂「章节来源」(精确到行号):
```
任务服务负责任务模板管理与执行「来源」[service/task/task_service.go#L23-L87](file://service/task/task_service.go#L23-L87)
```

**嵌真实代码片段**(关键逻辑,5-15 行,后挂 cite):
````markdown
```go
func (task *FollowTwitterTask) Verify(...) (...) {
    snsInfo := userInfo.GetSnsByType(utils.SNS_TYPE_TWITTER)
    ...
}
```
````

AI 推断(无直接源码证据)打 `[推断]`,不挂假溯源。行号来自 tree-sitter AST(Tier 2)或源码直读,不靠猜。

## L0 全局总图(入口 INDEX.md)

1. 一句话定位。
2. **架构图(C4 Context)**:系统与外部(用户/前端/三方 API/区块链)边界。
3. **部署图**:部署拓扑(服务节点/DB/Redis/MQ/网关)。
4. **子系统依赖图**(graphviz):Service/Subsystem 节点 + calls/depends-on 边。
5. **跨子系统协作图**:核心业务如何跨域协作(如 创建活动→完成任务→发奖励)。
6. 技术栈、业务域划分。
7. 下钻入口(每个节点 → L1)。

## L1 概览(每个 Service/Subsystem)

1. 定位与边界。
2. **架构图(C4 Container)**:服务内容器(API 层/业务层/数据层/缓存/MQ)。
3. **组件协作图**:内部主要组件如何协作。
4. 对外接口清单(Endpoint:方法+路径+职责)。
5. 对外时序概览(选:2-3 条核心对外调用时序)。
6. 上下游、数据存储、配置关键点。
7. 下钻 → L2。

## L2 模块地图(每个 Service/Subsystem)

1. **模块依赖图**(tree-sitter):包级依赖,标热点。
2. 模块/包结构说明(每个顶层包一句话)。
3. **核心业务流程图(flowchart)**:2-5 条热点业务流程(如"任务验证""奖励发放"),含判定分支。
4. **核心业务时序图(sequence)**:这些流程的跨层调用时序(客户端→service→task→外部API→DB)。
5. **状态机(stateDiagram)**:有生命周期的实体(如任务状态 Ok/Issued/Fired/Doubtful/Deleted)。
6. 类图(选:核心类型关系)。
7. 下钻 → L3 / cite 到代码。

## L3 实现细节(按需,可独立或并入 L2)

1. 关键函数(类#方法)+ 嵌真实代码 + cite。
2. **详细时序图**:关键函数内部调用时序。
3. 数据表关系 + 关键字段。
4. 异常处理/重试/容错(含错误码)。
5. 性能敏感点(缓存/批处理/异步/分布式锁)。
6. 测试覆盖关键用例。

## 模式差异速查

| 维度 | 多仓库 | 单仓库 |
|---|---|---|
| L0 节点 | 各仓库/服务 | 顶层子系统/领域 |
| L0 图 | 跨服务依赖+协作+部署 | 子系统依赖+协作+部署 |
| L1 对象 | 每个服务 | 每个顶层子系统 |
| INDEX.md | L0 跨服务总图 | L0 顶层子系统总图 |
